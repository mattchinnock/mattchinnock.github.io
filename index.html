    Pythio - Customizable Telemetry Solutions

    Pythio specializes in bespoke hardware monitoring, allowing companies to remotely manage and diagnose their hardware around the world.

    Project Description

    The platform manages thousands of devices, each with multiple sensors sending data back to a centralized server through WiFi or the GSM network. Data is stored in a MongoDB database, fed to an EXT JS frontend over websockets and parsed by visualization libraries such as highcharts.js. Pythio allows it's customers to see data about their machines through a web interface as if they were standing in front of it, from anywhere in the world.

    Role

    Full stack developer, UI designer. Building Javascript components to display and modify device data in real time via a custom Java application server. These components have to not only talk with the noSQL database, but also issue commands directly to the physical devices.

    Valence - Salesforce Integration

    Valence is an integration framework built natively in Salesforce. It radically reduces the time needed to build integrations between Salesforce and other systems, and empowers Admins when it comes to data synchronization.

    Project Description

    A large part of my services work at KapuHonu invloved connecting systems together. Valence is an incarnation of the experience gained from building these custom intregrations over and over. It provides an extensive and extensible platform to help admins, developers and execs connect their Salesforce orgs to any REST based system without having to spend the time, money and effort that comes with a completely bespoke solution.

    Role

    As co-founder, my role in the company was to steer user-focused and technical decision making, build relationships and help bring the product to market. As head of UI design, I'd craft user experience. As lead frontend developer, building responsive, functional Lightning components.

    Lead Marketplace*

    Lead Marketplace* is a Salesforce based app that collects, sanitises and standardises Leads (information about prospective customers). This data is initally gathered from multiple sources in varying formats, filtered, cleaned up and offered to the end user as a servicable group.

    Project Description

    Built using the robust Data Model of Salesforce, the marketplace utilises external integrations with systems such as Heroku and Elesticsearch for large scale data computation that standarises the data. Once Leads have entered the system, the end user has a myriad of tools available to view, edit and manage the Leads. The Lead aquisition process is recorded at every step, offering performance analysis and process efficiency improvements at every interaction.

    Role

    As a developer on this project I was one of two responsible for the UI design. On top of that, writing Visualforce and Apex code providing the frontend with real-time updates to Lead information through Dashboards, charts and cards. I also implemented a CTI (telephony), allowing users to call Leads directly from their record in the application through a custom built Softphone.
